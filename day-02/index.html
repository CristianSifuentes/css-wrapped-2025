Dialog Light Dismiss

Bringing a nice Popover API feature to <dialog>.

👍 66 · Limited availability
Try the demo · MDN · <dialog closedby>

One of the nice features introduced by the Popover API is the light dismiss behavior of popovers. This lets users close a popover by clicking outside of it — on the ::backdrop — or by pressing the Esc key.

From Chrome 134, this light dismiss behavior is also available on <dialog>, through the new closedby attribute, which controls the behavior:

closedby values

closedby="none"
No user-triggered closing at all. This is the default.
closedby="closerequest"
Pressing Esc (or another close trigger) closes the dialog.
closedby="any"
Clicking outside the dialog, or pressing Esc, closes it — like popover="auto".
<dialog closedby="none">…</dialog>        <!-- default: only a close() call can close it -->
<dialog closedby="closerequest">…</dialog> <!-- Esc closes it -->
<dialog closedby="any">…</dialog>         <!-- Esc or a click on the backdrop closes it -->

Try it live

Open each dialog below, then try pressing Esc and clicking on the dimmed backdrop to feel the difference between the three closedby values.

Checking native support…

closedby="none"

Esc and backdrop clicks do nothing here — this is the default behavior. You must use the button below.

closedby="closerequest"

Press Esc to close me. Clicking the backdrop won't do anything.

closedby="any"

Press Esc or click outside this dialog — on the backdrop — to close it.

closedby has limited browser availability. This demo detects support and falls back to manually wiring Esc/backdrop-click handling with JavaScript when the attribute isn't recognized, so the behavior above still works everywhere.
