scrollIntoView() container

Sometimes, scrolling only the nearest ancestor scroller is all you want.

👍 22 · Limited availability
Try the demo · MDN · scrollIntoView() container

The container option for Element.scrollIntoView() lets you perform a scrollIntoView that only scrolls the nearest ancestor scroll container. This is extremely useful if you have nested scroll containers. With the option set to "nearest", calling scrollIntoView won't scroll all of the scroll containers between the element and the viewport — just the closest one. 

slideList.addEventListener('click', (evt) => {
  // scrollIntoView will automatically determine the position.
  evt.target.targetSlide.scrollIntoView({container: 'nearest', behavior: 'smooth'});
});

Without container: "nearest", a click can drag this whole box up or down even though you only meant to scroll the carousel sideways.

scrollIntoView({ container: "nearest" }) has limited browser availability. This demo detects real support with a tiny hidden probe at load time; where it isn't supported, the "nearest" toggle falls back to manually scrolling just the carousel track, so you can still see the intended effect. With the toggle off, the demo always uses the standard, universally supported default scrollIntoView() — which is exactly what causes the outer box to jump.
