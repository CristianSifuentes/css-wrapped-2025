Customizable select

You can finally style HTML <select> elements with CSS.

👍 192 · Limited availability
Try the demo · Chrome for Developers · MDN · Customizable <select>

The time has finally arrived: you can now fully customize the HTML <select> element using CSS! To get started, apply the appearance: base-select CSS property to your <select> element — and to its ::picker(select). This switches it to a new, minimal state that's optimized for customization. 

select,
::picker(select) {
  appearance: base-select;
}

Using base-select unlocks several powerful features, including complete CSS customization. Every part of the select — the button, the dropdown list, and the options — can be styled with CSS: colors, fonts, spacing, even animations.

What comes with base-select

::picker(select)
Rendered in the top-layer, so it's never clipped by parent containers. The browser handles positioning and flipping automatically.
Rich <option> content
<img> and <span> can live inside options — flag icons, avatars, multi-line rows. No interactive elements like links, though.
<selectedcontent>
Reflects the selected option's HTML. Setting display: none on parts of it inside selectedcontent lets the button show only a subset — an icon, a name — while the option shows everything.

3. <selectedcontent> hides the description

selectedcontent .description {
  display: none; /* keep the description in the option, hide it in the button */
}

appearance: base-select has limited browser availability. In browsers without support, every <select> above still works — it just renders as a plain native dropdown, since the extra markup (the wrapping <button> and <selectedcontent>) is simply ignored.
